Note by Alan Rector:

The whole point of separating the linearization layer from the categorial layer is that

notions such as preferred parent only relevant to a linearlzation.

Clearly there has to be a "prime linearization" that provides the backwards compatiblity

with ICD-10 (and ICD-x). (Presumably the mortality linearization corresponding to the

existing ICD-10) But it should be a preferred linearization, not a preferred parent within

the categorization layer.

It is a regrettable necessity of the need for backwards compatibility that the external

identifiers will have to be derived from the prime linearization.
